Enterprise Market Intelligence Copilot

A local-first LLM analyst that answers enterprise market questions mixing hard pipeline numbers (TAM, SAM, MRR) with qualitative context on competitor moves and regulatory shifts — no cloud APIs, everything runs on-device.  ·  GitHub repo →

Architecture

  • Agent core: a LangGraph ReAct agent that decides per question whether to query structured data, search unstructured context, or chain both.
  • Tool 1 — SQL generator over a SQLite schema holding the hard numbers: TAM, SAM, MRR, pipeline figures.
  • Tool 2 — semantic search over a ChromaDB vector store (all-MiniLM-L6-v2 embeddings) holding qualitative material on competitors and regulation.
  • Serving: FastAPI deployment, models served locally via Ollama.

The debugging story: when the model avoids its own tools

v1 ran on Llama 3.1 (8B) and hallucinated numbers. Tracing the agent's behavior revealed a tool-avoidance pattern: rather than calling the SQL tool, the model was fabricating TAM and MRR totals directly in its answers. The fix wasn't a bigger prompt — it was switching to Gemma, whose native function-calling made tool use the path of least resistance. The made-up numbers stopped, and the agent began chaining SQL and vector search reliably.

The lesson: LLM hallucination is often an interaction failure between model and scaffolding, not raw model weakness. Diagnosing it means reading traces, not just outputs.

Making local-first actually reproducible

  • Diagnosed a Metal Performance Shaders (MPS) deadlock in the HuggingFace embedding pipeline on Apple Silicon and pinned embeddings to a CPU backend — fixing the hang and making the FastAPI deployment reproducible across machines.
  • Moved prompt-level guardrails into the system structurally: the prebuilt ReAct node plus per-tool schema docstrings, rather than fragile instructions buried in a mega-prompt.
